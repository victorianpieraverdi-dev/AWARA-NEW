# 🜔 Восхождение по линзам — мастер-план

Создано: 2026-06-17 · Статус: черновик ТЗ (ждёт решений по открытым вопросам)

Это корневой документ большой разработки «Линза как универсальный инструмент восхождения». Он собирает всю программу в один скелет, на который опираются все подзадачи. Связан с: [[wiki/synthesis/game-vision|Видение игры]], [[wiki/synthesis/agent-matrix-connections|Связи Агентов и Матриц]], [[wiki/synthesis/loka-to-chakra-mapping|Связь Лок и Чакр]], [[wiki/synthesis/card-rarity-hierarchy|Иерархия карт]].

---

## 0. Суть в одном абзаце

У нас есть единый «позвоночник» восхождения (9 чакр-мер + 14 лок + 21 агент). Каждая из 33 Матриц Восприятия — это отдельная «вселенная/линза», говорящая на своём культурном языке. Задача: в каждой линзе воспроизвести один и тот же скелет восходящего сознания, но в её родных образах, мифах и методах. А затем сделать так, чтобы при выборе линзы весь интерфейс и способ игры перестраивались под метод восхождения этой традиции. Линза = универсальный, но индивидуальный инструмент трансформации.

---

## 1. Позвоночник (канон — неизменяем)

- **9 чакр-мер** (вертикаль сознания): Муладхара → Свадхистана → Манипура → Анахата → Вишудха → Аджна → Сахасрара → Монада → Абсолют. Это «девятимерность + семичакровость».
- **14 лок** (территория восхождения): 7 верхних (Бхур → Сатья) + 7 нижних (Атала → Патала). См. [[wiki/synthesis/loka-to-chakra-mapping|Связь Лок и Чакр]].
- **21 Космический Агент** — покровители/силы на ступенях.
- **33 Матрицы** — линзы-вселенные.
- **693 соответствия** (21×33) — уже есть в кодексе (`matrices.json`, `agent_matrix_map.json`).
- **63 карты** (21×3).

Вывод: канонический «контур восходящего человека» = маршрут по 9 чакрам-мерам сквозь 14 лок под покровительством агентов. Все линзы переписывают ЭТОТ контур своими символами.

## 2. Рабочие потоки (7 направлений)

### WS1. Архетип восхождения в каждой матрице — «скелет восходящего сознания» ×33
Для каждой из 33 линз создать структурированный «контур восхождения»: как именно эта традиция описывает путь снизу вверх по 9 ступеням. Нужно глубокое исследование (историческое, мифологическое, энергетическое); где источников мало — аккуратно до-придумать в каноне.
Результат каждой линзы:
- родное имя пути (Дао, Древо Сфирот, Восьмеричный путь, Руническая спираль, путь Дуата и т.п.);
- 9 ступеней, спроецированных на 9 чакр-мер (родной символ + образ + метод/практика + теневое→дар + агент-покровитель + порог Света);
- образ «восходящего человека» этой вселенной (визуальный архетип);
- метод/способ восхождения (что игрок реально ДЕЛАЕТ в этой линзе).

### WS2. Адаптивный интерфейс под линзу
При выборе линзы UI перестраивается под её метод: палитра, символы, лексика, сам способ практики на экране Тигля/Результата. Линза становится «режимом игры», а не просто фильтром текста.

### WS2+. Интерферизация и скрещивание линз (до 4)
Каждая линза — свой дизайн/режим UI. Линзы можно скрещивать, до 4 сразу:
- 1 линза → интерфейс целиком на её языке (эталон — Ведическая).
- 2–4 линзы → элементы, расчёты и интерфейс комбинируются: добавляются вычисления каждой матрицы, тексты и символы звучат на языке всех выбранных линз.
- Режим до 4 линз — для подготовленного сознания игрока (продвинутый уровень), открывается по росту Света (связь с WS4 и WS5).
Технически: awara-lens-mode.js держит активный набор 1–4 линз; рендер = наложение слоёв (позвоночник + N родных языков), расчёты = объединение формул матриц.

### WS3. Ансамбль 5–6 эзо-ИИ под одной маской
5–6 ИИ-голосов с эзотерическим уклоном (например: толкователь-символист, мистик-наставник, хранитель лора, диагност состояния, мифо-рассказчик, детский голос) объединяются под одним «уникальным ИИ игры». Снаружи — один характер; внутри — маршрутизация на нужного «специалиста».

### WS4. Образ игрока (персональный помощник)
Профиль игрока, который копит паттерны: выборы линз, ритмы, повторяющиеся темы, прогресс по Свету. На его основе помощник подстраивает советы и интерфейс. Персонализированный «спутник».

### WS5. 12 нисходящих + 3 восходящие линии (посвящения)
Модель из 15 линий мистических инициаций. 12 нисходящих (воплощение/проявление) + 3 восходящие (возврат к Истоку). Открываются за определённое количество Света — это «гейты Света» поверх экономики Света.

### WS6. Карта промтов (ждём файл от автора)
Автор пришлёт «карту промтов». Задача: понять её структуру и грамотно встроить — как слой данных, который кормит и эзо-ИИ (WS3), и генерации даров, и тексты ступеней.

### WS7. Детская версия (бэклог — ПОСЛЕ взрослой)
После готовности взрослого приложения — адаптация на детский уровень: упрощённые образы, мягкая лексика, игровая подача того же скелета. Зафиксировано как отдельная задача.

---

### WS8. Грок-гейтинг генераций
Генерации (музыка/видео/3D/образы) открываются постепенно по уровню грока: на ур. 1–2 — 1–2 генерации, дальше больше. Панель Тигля (9 блоков) = шаблон вывода. См. [[wiki/synthesis/sostoyanie-igry-generacii-2026-06-17|Состояние игры]].

### WS9. Маскированные ИИ + 3D-аватар + личная Вселенная
Маскированные ИИ-помощники (связка с WS3), видео + 3D-анимация аватара, персональная «реальная Вселенная» под игрока, расширение натальных/игровых Вселенных с замаскированными отсылками к источникам.

### WS10. Слой «состояние игры»
Основа — суточный планёр 24 часа (цветовые часы/энергии), запись состояний в ячейки, образ дня → супер-образ недели → супер-энергии → строительство своей вселенной. Цвет/вкус/звук-таблица, стрики, инерция практик, Выполнение = Свет. Новый агент: Строительный «МасонХ». См. [[wiki/synthesis/sostoyanie-igry-generacii-2026-06-17|Состояние игры]].

## 3. Предлагаемая модель данных (на обсуждение)

Новый файл `data/ascension.json` — по одной записи на матрицу. Скелет одной линзы:

```
{
  "matrix": "vedic",
  "path_name": "Лестница Лок",
  "ascending_human": "образ/архетип восходящего",
  "method": "что игрок делает в этой линзе",
  "steps": [
    {
      "n": 1,
      "chakra": "muladhara",      // привязка к 9 чакрам-мерам
      "loka": "bhur-loka",         // привязка к 14 локам
      "native_name": "родной термин ступени",
      "image": "образ/символ",
      "practice": "метод/действие",
      "shadow": "тень",
      "gift": "дар (тень→дар)",
      "patron_agent": "prithvi",   // из 21 агента
      "light_threshold": 0          // порог Света для открытия
    }
    // ... 9 ступеней
  ]
}
```

Плюс `data/initiations.json` для WS5 (15 линий) и расширение профиля игрока в `STATE` для WS4.

## 4. Архитектура движка (черновик)

- `ascension.json` + `awara-ascension.js` — загрузка контуров и рендер «лестницы линзы».
- `awara-lens-mode.js` — перестройка UI под выбранную линзу (палитра/лексика/метод). Слой поверх движка, ядро не трогаем.
- `awara-oracle.js` (WS3) — фасад одного ИИ + внутренняя маршрутизация на 5–6 ролей; надстройка над `aiSystem()`/`aiCall()`.
- `awara-player-image.js` (WS4) — сбор паттернов в `STATE`, выдача персональных подсказок.
- `awara-initiations.js` (WS5) — 15 линий, гейты по Свету.
- Карта промтов (WS6) — отдельный слой данных, который читают oracle и генераторы даров.

Инварианты (НЕ трогать): `MATRIX[k][1]`, `window.POLAR`, русские i18n-ключи, antique_greco_roman (#17). Любой новый модуль — навесной слой, как уже сделаны tigel-live / daimon-live / result-clean.
