# Решение по лобби-модулям — 2026-06-02

**Статус:** закрыто (решение принято, вопрос больше не висит).

## Контекст
После завершения безопасного распила (7 блоков вынесено в файлы) оставался один пункт —
«миграция инлайн-лобби на ES6-модули» (`lobby-module.js` / `menu-module.js` и цепочка
`core/state/ui/canon/game-module.js`). При разведке выяснилось, что это **не вынос инлайна**,
а миграция на параллельную, незаконченную архитектуру.

## Найденные мины
1. **Разный ключ состояния.** Модули → `awara_v255_state`, живая игра → `awara_v258_state`.
   Подключение как есть **обнулило бы лобби** и раздвоило прогресс.
2. **Модули никогда не подключались** — в `core-module.js` прямо написано «будет в T-058».
3. **canon-module.js тянет `data/*.json`** — без этих файлов канон падает.
4. **Канон расходится** — в модуле `CARDS_COUNT=1578`, в проекте 63/693.

## Решение
- **Инлайн-лобби в `index.html` = КАНОН.** Источник истины — он.
- ES6-модули — **отложенный эксперимент**, в рабочую игру не вшиваем.
- Файлы НЕ перемещали (нельзя проверить импортёров на других страницах без полнотекстового
  поиска — перенос мог бы тихо сломать другие `.html`). Вместо этого решение зафиксировано
  документально: `C:\AWARA\js\_MODULES-STATUS.md`.

## Если когда-нибудь делать миграцию (отдельный проект)
1. Свести state на `awara_v258_state` + адаптер форматов.
2. Обеспечить `data/*.json` или переписать `canon-module` на инлайн-канон.
3. Добавить `data-*`-хуки в разметку лобби.
4. Подключить модули + развести дубли функций.
5. Тест через `js/test-lobby-module.html`.

## Итог распила
Безопасная реконструкция полностью завершена. Лобби-миграция — осознанно отложена, а не забыта.
