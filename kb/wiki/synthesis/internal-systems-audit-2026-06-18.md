## 1. Идея

В матрице **#14 (генные ключи)** развёрнута глубокая «внутренняя решётка»:

- 64 ключа → **21 кольцо кодонов** (аминокислоты) → привязка к **21 агенту**;
- слои: **64 минерала**, **3-уровневые тотемы** (птица / зверь / насекомое-рыба);
- полярность Сиддхи ⇄ Дар ⇄ Тень.

Файл-спутник: `data/matrices/gene_keys_living_library.json`. Фиксированный позвоночник 21 агента при этом **не менялся**.

**Цель:** сделать такой же спутник для КАЖДОЙ традиции, у которой есть аутентичная внутренняя система. Не трогаем фиксированный позвоночник 21 агента и матрицу #17 (antique_greco_roman).

---

## 2. Критерий «подобной системы»

Система достойна отдельной living-library, если есть:

1. **фиксированная нумерованная решётка** (64 / 27 / 99 / 256 / 22 …);
2. **слои соответствий** на каждый элемент (божество / символ / буква / планета);
3. бонус — **тотемный слой** (животное / растение / камень), как в генных ключах.

---

## 3. Универсальная модель внедрения

### 3.1 Распределение по позвоночнику (21 агент)
Любую систему из N элементов раскладываем на **21 группу — по одной на агента** — по архетипическому резонансу агента (стихия / гуна / луч / светило / васту / домен), ровно так, как 64 ключа легли в 21 кольцо кодонов. Количество на агента **неравномерно** (как в кольцах: 1-6 ключей). Это «компас, а не рельсы».

### 3.2 Уровни (слои) каждого элемента
- **L1** имя / символ (en + ru)
- **L2** полярность: тень ⇄ дар (как Сиддхи ⇄ Тень)
- **L3** тотем: животное / растение
- **L4** минерал / камень
- **L5** буква / число / планета / орган (если есть в традиции)

Заполняем только те слои, что аутентичны; отсутствующие опускаем (без выдумок).

### 3.3 Схема файла `*_living_library.json`
```
{
  "system_slug": "...",            // напр. iching_64
  "tradition_matrix": "#N slug",   // напр. #27 chinese_iching
  "system_name": "...",            // напр. 64 hexagrams
  "element_count": N,
  "spine_distribution_rule": "N -> 21 agent by domain resonance",
  "layers": ["symbol","polarity","animal","mineral","letter"],
  "reconstructed": false,
  "spine": {
    "<agent_slug>": { "ids": [..], "note": "..." }
  },
  "items": [
    {"id": N, "name_en": "...", "name": "<\\u ru>", "agent": "<slug>"}
  ],
  "sources": []
}
```

### 3.4 Инварианты
- Позвоночник 21 агента **фиксирован** (21 slug не меняется).
- **Без выдумок:** только аутентичные источники; модерн/реконструкцию помечать `"reconstructed": true`.
- Кириллица в JSON-данных пишется через **\u-эскейпы** (ASCII) → верификация чтением, скан на битые символы.
- Спутник — **отдельный файл**; #14 не трогаем; **#17 НЕ трогаем**.
- Спутники кладём в `data/matrices/<slug>_living_library.json`.
