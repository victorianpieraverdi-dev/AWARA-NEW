# Два сайта AWARA и план поэтапного переноса

> Зафиксировано 2026-06-02, ночь. Решение Павла: «работаем поэтапно, старое не уничтожать — пусть остаётся эталоном для переноса, сохранить всё в Obsidian и в память».

---

## Главное (суть в одном абзаце)

Сейчас у проекта фактически **два разных сайта**:

1. **`index.html`** — НАСТОЯЩАЯ игра. Весь готовый контент и механики: вселенная, квесты, натальная карта, гримуар, тигель, оракул, аюрведа, банк, сферы посвящения, рич-лобби-мандала (RA в центре). ~1.16 MB, >16000 строк, монолит.
2. **`lobby.html`** — НОВЫЙ модульный каркас. Красивая обложка-мандала (сделана под вид `index.html`), но **внутри пусто**: разделы (ИГРА/ХРОНИКА/ОБМЕН/ОБЩЕНИЕ) ведут на заглушки (меню-модалка, тосты «скоро»), а не на реальные экраны.

**Решение:** `index.html` остаётся боевой игрой и эталоном. **НИЧЕГО СТАРОГО НЕ УДАЛЯЕМ.** Ссылку «войти →» в `entry.html` НЕ переключаем на `lobby.html`. Модули развиваем поэтапно рядом, без риска для боевой игры.

---

## Почему это два разных сайта (чтобы не забыть)

| | `index.html` (старый / боевой) | `lobby.html` (новый / модульный) |
|---|---|---|
| Статус | Полноценная игра | Пустой каркас + красивый вид |
| Контент | Всё есть | Нет (только свет/ранг/агент из state) |
| Код | Монолит 16k+ строк | Чистые ES6-модули |
| Разделы ведут | На реальные экраны | На заглушки «скоро» |
| Роль | Боевой + эталон | Полигон архитектуры |

Оба работают на одном живом состоянии `awara_v255_state` (один localStorage-ключ), поэтому свет/ранг в лобби показываются правильно. Адаптер `state-module.js` пишет **merge-safe** — не затирает чужие ключи (`sphereData` и др.).

## План поэтапного переноса (без разрушения старого)

Принцип: каждый экран из `index.html` переносится в модуль ПО ОДНОМУ, сверяясь с оригиналом как с эталоном. Оригинал живёт и работает всё это время.

### Очередь экранов на перенос (из меню/разделов игры)
- [ ] **ИГРА → Пространства**: сферы, вселенная, земля, творение
- [ ] **ХРОНИКА**: летопись / коллекция карт / путь
- [ ] **ОБМЕН**: банк света / дары
- [ ] **ОБЩЕНИЕ**: союзы / социальное
- [ ] Натальная карта (калькулятор)
- [ ] Гримуар (24/33 матрицы)
- [ ] Тигель (дневник/записи)
- [ ] Оракул
- [ ] Аюрведа / доши
- [ ] Паспорт души (расширить)

### Правила переноса
1. Сначала описать экран в Obsidian (как выглядит, что делает) — на случай если забудется.
2. Перенести в модуль и подключить на отдельной test-*.html.
3. Проверить в браузере.
4. Только после полного паритета всех экранов — рассматривать переключение entry→lobby.
5. `index.html` НЕ удаляем никогда, пока не будет полной замены и явного окея Павла.

## Ключевые тех-факты (чтобы не потерять)
- Живой ключ состояния: **`awara_v255_state`** (НЕ v258 — тот возвращал null). В нём `totalLight`, `sphereData`.
- Другие ключи: `awara_patch_v258`, `awara_sphere_entries`, `awara_initiation_v1` (сферы shiva/vishnu), `awara_matrix=neutral`, `awara_lang=en`, стрики, инвентарь.
- Канон неприкосновенен: 21 / 33 / 14 / 9 / 693.
- Окружение: localhost:8765 ≠ github.io — это разные хранилища, основной прогресс не затронут.
- Репо: github.com/victorianpieraverdi-dev/awara-game ; живой: victorianpieraverdi-dev.github.io/awara-game/
- Монолит нельзя редактировать хирургически (нет grep / range-read в MCP). Поэтому модульный путь идёт «рядом», а не внутри монолита.
