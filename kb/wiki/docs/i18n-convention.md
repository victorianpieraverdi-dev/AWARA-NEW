# Правило: всегда RU + EN переводы

> Постоянное правило проекта AWARA. Введено 2026-06-04 по запросу владельца.
> Действует ВСЕГДА, для любой новой функции / экрана / надписи.

## Суть

Любой новый текст, который видит игрок, добавляется СРАЗУ на двух языках — русском и английском. Не «потом доперевожу», а в том же шаге, что и сама фича.

## Где это живёт

- Файл переводов: `js/i18n.js`
- Структура: `translations[lang][key] = value`, где `lang` = `'ru'` или `'en'`
- Язык хранится в `localStorage.awara_lang` (по умолчанию `'en'`)
- Переключатель RU/EN есть в интерфейсе (тоггл в лобби и на экранах)

## Чек-лист при добавлении любой надписи

1. Придумать осмысленный ключ в стиле существующих: `<секция>.<смысл>` (например `lobby.enter`, `egg.help.title`).
2. Добавить значение в блок `ru: { ... }`.
3. Добавить ТО ЖЕ ключ в блок `en: { ... }` с английским переводом.
4. В разметке/коде использовать ключ через систему i18n, а не хардкод строки.
5. Проверить переключение RU↔EN — обе версии читаются и не ломают вёрстку.

## Особый случай: 3D-надписи (lobby-3d.html)

В 3D-лобби подписи сфер/панелей рисуются через CanvasTexture (`makeOrbLabel`, `makePanelTexture`). Они НЕ обновляются автоматически при смене языка. Поэтому:

- Тексты 3D-надписей тоже берём из i18n (а не хардкодим).
- При переключении языка нужно перерисовать CanvasTexture (перегенерировать текстуру орбов/панелей).
- Пока эта перерисовка не реализована — это известный TODO; новые 3D-надписи всё равно заводим сразу с RU+EN ключами, чтобы при включении перерисовки всё заработало.

## Коротко

**Нет английского ключа = функция не считается готовой.**
