# Прокачка Линз: уровни, чёткость, темы интерфейса и Колесо Тигеля

Создано: 2026-06-14 · Источник: голосовой поток (Charvet/Pavel, 14.06.2026)
Статус: концепт для проработки

> [!info] Контекст
> «Линзы» = **Матрицы Восприятия** (33 по канону). Механика добавляет к глобальной прогрессии ([[wiki/mechanics/grayscale-to-color|ч/б → цвет]], renderTier) **второе, индивидуальное измерение**: какие матрицы игрок проживает чаще, те сильнее формируют его «духовное Я» и внешний облик интерфейса.

---

## 0. Базовый поток (UX-ядро)
Цель: «**Зашёл → прожил → выбрал линзы → получил ответ → дальше**». Один главный жест, минимум кнопок.

- **Тигель** — главный экран действия; «**Исток**» = текущее состояние/момент игрока (то, что происходит сейчас).
- Шаг 1 — **Захват дня**: голос/текст, одно касание (как voice в мессенджере).
- Шаг 2 — **Выбор до 3 линз**: вручную ИЛИ через Колесо Тигеля (рандом, см. §4).
- Шаг 3 — **Ответ через линзы**: толкование дня сквозь выбранные матрицы (Оракул/Даймон).
- Шаг 4 — **Задание дня** от доминирующей линзы (§3) → «Закрыть».
- Кнопки: крупные тач-зоны (≥48px), приклеены к низу (зона большого пальца), переключение свайпом. Нижняя навигация ужимается до 3 зон: **Тигель** (действие) · **Путь** (прогресс/портрет) · **Хроника** (история).

---

## 1. Уровни и Чёткость линзы
- Каждая матрица копит опыт: проживание через неё = +использование/опыт.
- Уровни = **Чёткость**: `Тусклая → Проявленная → Чёткая → Гранёная → Сияющая`.
- XP даёт не только выбор линзы, но и **выполнение её задания** (§3) — чтобы Чёткость означала практику, а не спам-клики.
- Чем выше Чёткость — тем ярче сама линза в UI и тем сильнее её влияние на интерфейс (§5).

---

## 2. Духовный портрет (индивидуальность пути)
- Частота линз складывается в **спектр/радар** (напр.: Шаманская 15, Адвайта 10, Шинто 8…).
- Это «характер индивидуального духовного Я». Живёт на экране **«Путь»** (или внутри Паспорта Души).
- Портрет = персональный профиль, влияющий на карту дня, задания и Даймона.

---

## 3. Задания от линз
- У каждой матрицы уже есть quest-данные ([[wiki/entities/_matrices_summary|матрицы]] · `entities/quests/*`, поле `questStyle`).
- Доминирующая линза дня выдаёт **задание в своём стиле** (у Шаманской — journey / shapeshift / heal / retrieve / dance и т.д.).
- Выполнил задание → XP линзе → растёт Чёткость → растёт влияние на интерфейс. Это замыкает петлю «практика → прокачка → облик».

---

## 4. Колесо Тигеля (рандомный выброс 3 матриц)
- На входе/на Тигеле — **колесо с глифами линз** по кругу. Тап в центр → вращение → выбрасывает **3 случайные** матрицы.
- Две кнопки: **«Выбрать»** (вручную) и **«Доверить Тигелю»** (рандом).
- Режим рандома на выбор: **чистый случай** («воля судьбы») / **взвешенный** (чаще подкидывает редко используемые линзы — расширяет путь).
- Визуал: золотое вращение, приземление со «вспышкой» — переиспользовать sparkles из FX-слоя (`awara-fx`).

---

## 5. Темы интерфейса от линз («скины как в CS») + смешивание
Расширяет [[wiki/mechanics/grayscale-to-color|renderTier]] и идею «Супер Ап» из [[wiki/synthesis/ideas-backlog-2026-06-11|бэклога идей]] (системы 1/2/7): глобальный tier = *насколько* проявлен мир, **тема линзы = в каком стиле** он проявлен.

- Каждая матрица = **визуальная тема**: палитра, текстуры, движение, иконки, звук.
- **Разблокировка по Чёткости**: линза дошла до уровня N → кнопка «Сменить облик» → весь интерфейс перекрашивается под неё.
- **Тиры внутри линзы**: выше уровень → богаче тема (минимализм → роскошь: больше частиц, градиентов, анимации, сакральной геометрии).

### Смешивание (Адвайта + Тантра + Шинто)
- «**Алхимия облика**»: при 2+ открытых темах их можно смешивать.
- 3–5 пресетов на линзу + **микшер**, где веса = относительные уровни линз игрока → интерфейс буквально отражает **состав** его пути. Затем можно подкручивать вручную.

## Техническая реализация (ложится на текущий код)
- Темы = наборы **CSS-переменных** (как уже в `awara-ds.css` / `awara-fx.css`): «тема» = значения токенов + опц. FX-модули.
- Переключение: атрибут `data-lens-theme="advaita"` на body/.phone (по аналогии с `data-theme="level0..4"` из grayscale-to-color).
- Смешивание = взвешенная интерполяция палитр (HSL-mix) + наслоение FX-модулей по весам уровней.
- Состояние линз в PlayerState: `lenses[id] = { uses, xp, clarity, themeTier }`.
- Колесо: случайный выбор из доступных матриц (опц. вес = 1/uses для редких).
- **Тач вместо hover (мобайл):** блик следует за пальцем (pointermove на тач), карта «зажигается» по тапу, у выбранной линзы свечение постоянно; опц. параллакс по гироскопу.

## Открытые вопросы
- Темп роста Чёткости (10–15 проживаний за месяц → какой уровень?).
- Сколько тем-пресетов на линзу для MVP (1 vs 3–5).
- Колесо: взвешенный рандом vs чистый случай (или тумблер).
- Где живёт «Духовный портрет» — отдельный экран «Путь» или часть Паспорта Души.
