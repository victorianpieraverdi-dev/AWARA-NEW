## 2. Целевая структура по экранам

### 2.1 ИСТОК (`s-istok`)

**Цель:** «Кто я сегодня в пути» — ежедневный снимок, не музей.

| Блок (сверху вниз) | Что должно быть | Источник данных |
|--------------------|-----------------|-----------------|
| **Sticky-хедер** | Активные матрицы, подсказки, язык | STATE.mats, i18n |
| **Индикатор света / осознанности** | Частицы сегодня, свет, стихии (уже есть) | particles, axes, energy |
| **Духовный портрет** | Уровни и качества **линз**, возможность перехода в линзу, связь со стихиями | lenses, ascension, matrices |
| **Хроника пути** | Наполнение под **ежедневную** игру (не абстрактный лор) | chronicle / day log / matQuests done |
| **Миф** | Ниже индикатора (поменять местами с индикатором, если сейчас наоборот) | lore / engine texts |

**Задачи:**
1. Поменять местами: **индикатор вверх, миф вниз**
2. Переписать **Хронику пути** под ежедневный цикл (Ключ Дня → квесты → свет → оракул)
3. **Духовный портрет** = линзы формируют портрет (+ стихии из индикатора)
4. Sticky верхних окон при скролле

**Связь с Roadmap v3:** п.10 Дневник/итоги, п.08 Профиль (частично), п.05 Свет.

---

### 2.2 ТИГЕЛЬ (`s-tigel`)

**Цель:** «Печь дня» — матрицы + актуальные задания.

| Блок | Что должно быть |
|------|-----------------|
| Sticky-хедер | Как на Истоке (общий компонент) |
| **Окна матриц** | Визуальная проработка (Linear/Aceternity), не «просто карточки» |
| **Задания от матриц** | Актуальные; недоступные **свёрнуты/скрыты** |
| **Будущие возможности** | Видно «что откроется», не кликабельно / серо |
| **Активация за частицы** | Если игрок может/хочет больше — добор слотов за ✧ |

**Перенос:**
- Задания **от локаций** → уходят в **План** (не дублировать в Тигле)

**Связь:** Квесты v2 (селектор, причины, арки L5+), Roadmap п.05.

---

### 2.3 ПЛАН (`s-plan`)

**Цель:** «Что я выбираю и что предлагает игра».

| Блок | Содержание |
|------|------------|
| **Выраженное намерение** | Игрок формулирует; ритуальный/майнд-мап слой (Roadmap п.01) |
| **Предложения игры** | После «узнавания» игрока (профиль, линзы, оси, натал) |
| **Задания от матриц** (сводка / ссылка) | Согласовано с Тиглем — не ломать единый day-state |
| **Задания от локаций** | Перенесены **сюда** из Тигеля, **под** матричными |
| **Цикл:** создаёт → реализует → бонусы | UX и начисление через particles/XP |
| **Кнопки-потоки** (Душа, Свет, Образ бытия, Распределение, Разлив) | Скорее **под капот** / advanced; на поверхности не шуметь |

**Связь:** Roadmap п.01 Намерение, п.08 Профиль, п.11 Синхронизация.

---

### 2.4 ДАЙМОН / АРДВЕН (`s-daimon`)

**Цель:** живой спутник — просто снаружи, глубоко внутри.

| Блок | Действие |
|------|----------|
| Качества Даймона | Пересмотреть из **новых данных** (`daimon-stages`, forms, engine) |
| Арка речи | Актуальные тексты / мена по матрицам |
| Натальная основа | Сложное **под капот**, снаружи 1–2 понятные строки |
| «Мой нрав» | Переработать формулировки |
| **Выбираемые грани** | Перенести/добавить **из Натала** сюда |

**Связь:** Roadmap п.07 Даймон глубина; FULL-CONTEXT §9 (4 стадии / 27 форм — сверить с кодом).

---

### 2.5 НАТАЛ (`s-natal`)

**Цель:** карта рождения — не дашборд всего мира.

| Действие | Детали |
|----------|--------|
| Полная переделка UX | Иерархия: ядро натала → раскрываемые блоки |
| Свернуть окна | Слишком много инфы «в глаза» |
| Грани | Часть уезжает в Даймон (выбираемые) |
| Под капот | Расчёты, сырые таблицы, вторичные аспекты |

**Связь:** Roadmap п.08 Профиль / Генные Ключи как грани.

## 3. Сквозные системы (не экраны)

### 3.1 Кейсы / рарность

| | |
|---|---|
| Концепция | Открытие **за частицы света** (пересмотр экономики в `engine_config.json`) |
| Содержимое | Фрагменты карт + будущие артефакты/бонусы |
| Где искать | `awara-rarity.js`, хроника / `app/istok.html` (продолжение мира), exports карт |
| UI | Блок на Истоке и/или после дня; не обязательно отдельный нав-таб v1 |

### 3.2 Милость / Дары Духа

| | |
|---|---|
| Суть | Нисхождение силы, уровни силы |
| Где | `milost.html`, `data/milost-sources.json` (7 источников / 7 Лучей) |
| Задача | **Найти в данных + явно показать в игре** (Исток и/или Даймон) |

### 3.3 Частицы ↔ Свет ↔ Структуры

| Термин | Роль (engine_config) |
|--------|----------------------|
| ✧ Частицы | Сырьё осознанности за действия дня |
| ☀ Свет | После ночной конвертации / прогресс |
| Световые структуры | Мера 4–9, строятся из **света** (6 структур) |

UI должен **не путать** частицы и свет (как в display-строках engine).

### 3.4 Sticky-хедер (общий)

Единый компонент/паттерн для Исток, Тигель, План, Летопись (минимум):  
активные матрицы · подсказка дня · язык · (опц.) ✧/☀.
