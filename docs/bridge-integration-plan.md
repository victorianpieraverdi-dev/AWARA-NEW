# AWARA · Мост тигля и большой игры — план интеграции

> Замысел соединения **маленькой игры (Тигель)** и **большой игры (AWARA)**: свет, рождённый из прожитого дня, со временем перетекает через обученный ИИ в сферы, стихии и космос игрока — с честной проверкой крупных генераций.

## Стратегия: песочница → слияние

- **Маленькая игра (Тигель) — песочница.** Нужна, чтобы дёшево перебирать варианты механик, прежде чем вшивать их в большой организм.
- **Большая игра (AWARA)** — конечная цель: свет уходит туда через обученный ИИ.
- **Порядок правильный:** сначала обкатать в тигле, потом строить мост.
- ⚠️ **Главная ловушка отложенного слияния:** песочница может уйти в сторону так, что потом не стыкуется.
- ✅ **Лекарство:** заранее договориться об одном общем формате обмена («пакет света»), даже если сам мост построим позже. Тестировать варианты *внутри* формата, не ломая стык.

## Контракт «пакета света»

Каждый прожитый в тигле день отдаёт наружу один и тот же объект. Пока он пишется в `localStorage`, позже — тот же объект уходит в большую игру.

```json
{
  "date": "2026-06-16",
  "light": 84,
  "elements": { "fire": 0, "earth": 0, "air": 0, "water": 0, "ether": 0 },
  "spheres": { "body": 0, "soul": 0, "earth": 0, "cosmos": 0, "universe": 0 },
  "lenses": [],
  "daimonTrust": 62,
  "honesty": 1.0,
  "proof": null
}
```

- **Ближние сферы (тело/душа) растут быстрее, дальние (космос/вселенная) — медленнее.** Короткая петля даёт мгновенную отдачу, длинная копит горизонт.
- Общий ключ хранения (например `awara.*`), чтобы и тигель, и 3D-движок читали одно состояние.

## ИИ-маршрутизатор света

ИИ превращает свободный текст переживания в структуру — «в какие сферы наливать, какие стихии, слова, триггеры».

- **Заполняет рубрику, а не пишет свободно:** сфера + стихия + интенсивность + тип переживания (творение / служение / тень / практика).
- **Объясняемость = доверие.** Игрок видит: «твоё переживание ушло в сферу Души, стихия Вода, +7, потому что…».
- Это прямая замена нынешнему `baseLightFromText` (regex по ключевым словам, который можно «нафармить») — честное, прозрачное прочтение вместо чёрного ящика.
- **Специальные фильтры** настраивают: какие слова/триггеры → какие сферы и стихии, как считается интенсивность.

## Proof-of-experience — проверка крупных генераций

Подтверждение, что за светом стоит реальный прожитый опыт. Потенциально — **фирменная идея проекта** (этого почти ни у кого нет).

- **Пропорциональность.** Маленькие ежедневные записи — доверие по умолчанию, без трения. На проверку (фото / видео / другие игроки) уходит **только крупное и высоконаградное**.
- **Репутация снижает трение.** Механику доверия Даймона поднять на уровень игрока: чем дольше человек честен, тем меньше доказательств с него спрашивают. Доверие как накопленный капитал.
- **Кто проверяет пиров.** Пиринговое подтверждение (как консенсус в блокчейне) красиво, но геймится (сговор). Нужны случайные проверяющие, ставка репутацией, порог «несколько независимых», а не простое большинство.
- **Приватность.** Личные фото/видео переживаний: наружу уходит только доказательство/хэш, не само содержимое.

## Принципы (чтобы не потерять суть)

- Свет = осознанность, не деньги. Светообмен — поле резонанса и служения, **не рынок**.
- Механика и метафора — один объект: тир меняет физику мира, свет меняет геометрию, невнимание гасит творение (Пралайя).
- Честность петли важнее богатства механик: защищать «ощущение возврата» и подлинность вклада.

## Что уже работает в Тигле (опора)

- **Настоящий натальный движок** (Кеплер, аянамша Лахири, накшатры) → **Даймон выводится из накшатры Луны** игрока.
- **Замкнутая петля дня:** запись дня → «Плавить» → Свет → Результат (совет + дроп по уровню Света: совет → миф → артефакт → трек → аюрведа → обложка) → «Прожито» → клетка в 365-дневной летописи + сброс дня, +доверие.
- **Ощущение возврата уже есть:** вспышка лучей/искр (`reward.js`) и видимое разложение Света по слагаемым (`advice-energy.js`).
- **33 матрицы-линзы** (макс. 3/день), энергии и полярности (свет ⇄ тень), покровитель дня по дашам.
- **ИИ-слой** (Даймон Ардвен, формы Лик ⇄ Зверь) — задел под будущий обученный ИИ света.
