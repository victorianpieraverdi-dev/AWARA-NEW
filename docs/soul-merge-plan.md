# AWARA · План сборки родной Души (React)

Дата: 2026-06-17
Решение: Душа в новой игре собирается из лучшего двух доноров. Старые HTML НЕ сносим — они остаются референс-донорами.

## Доноры

### СТАРАЯ — initiation-space.html («Карта Души» / Inner Channel Space)
- Canvas-2D. Сферы shiva (ОСНОВА) / vishnu (СЕРДЦЕ) / brahma (МЕЧТА) + PRIMAL (ДУША) + COTV.
- Экономика света: LIGHT{total, flow, exchanges, soulLight}; светообмен между сферами.
- Ритуал рождения сфер (birthThreeSpheres).
- Связи с большим миром: mind-space, temple, earth-player.
- Сейчас питается нашим светом через мост (window.AwaraSoulBridge).

### НОВАЯ — spheres-v2.html («Пространство Души v2»)
- Canvas-2D. Сферы: ДУША · ОСНОВА · СЕРДЦЕ · РАЗУМ · СВЯЗИ.
- Богатый слой данных на сферу — 5 вкладок: Записи (подсферы до 3, слоты открываются по свету), Журнал (типизир. записи + комменты), Планы/намерения, Медиа, Стиль.
- Онбординг-туториал (пошаговый, с точками прогресса).
- Spark-помощник (летающая искра + подсказки) и Spark-панель (ИИ-инсайт: insight + советы + вопрос).
- Вложения: голос/фото/видео/ссылки + соцсети (профиль).
- Хэндофф дня → Тигель (awara_soul_day_context).
- Кастомизация оформления (цвет/размер/свечение). Мобайл-first панель.
- localStorage: awara_tutorial_completed_v2, awara_subspheres_v3, awara_soul_day_context.

## Что берём откуда (лучшее из двух)

### ИЗ НОВОЙ (база UX и данных)
- Модель сферы с подсферами (до 3, слоты открываются по свету).
- 5 вкладок: Записи · Журнал · Планы · Медиа · Стиль.
- Онбординг-туториал.
- Spark-помощник и Spark-панель (ИИ-инсайт).
- Вложения (голос/фото/видео/ссылки) + соцсети.
- Хэндофф дня → Тигель.
- Кастомизация оформления. Мобайл-first панель.

### ИЗ СТАРОЙ (канон и свет)
- Архетипы/канон: shiva (Основа) / vishnu (Сердце) / brahma (Мечта) + PRIMAL (Душа) + COTV.
- Экономика света: total/flow/exchanges/soulLight, светообмен между сферами.
- Ритуал рождения сфер.
- Связи с большим миром: mind-space, temple, earth-player.
- Мифическая рамка / лор.

### ОБЩИЙ КЛЕЙ (наш слой)
- light-core (window.AwaraLight) — единый источник света; Душа читает НАШ свет, а не свой отдельный.
- tigelBridge — нормализованный свет 0..1.
- Принцип «компас, не рельсы»: подсветка advisory, не геймплейная стена.

## Целевая архитектура (React, app/soul/)
- SoulSpace.tsx — контейнер сцены (canvas-2D), монтируется в istok.
- soulModel.ts — типы Sphere/SubSphere/JournalEntry/Plan/Media + дефолтный канон (5 сфер с архетипами).
- useSoulState.ts — состояние + persist (ключ awara_soul_v1), миграция из awara_subspheres_v3.
- SoulPanel.tsx — панель сферы с 5 вкладками.
- soulLightBridge.ts — чтение из AwaraLight/tigelBridge, распределение света по сферам.
- soulSpark.ts — ИИ-инсайт через awaraAi (deepseek).
- soulTigelHandoff.ts — день → Тигель (awara_soul_day_context).

## Порядок сборки (инкременты)
1. soulModel.ts + дефолтный канон 5 сфер (архетипы из старой).
2. SoulSpace.tsx скелет: рисуем сферы на canvas, яркость от нашего света (мост).
3. useSoulState.ts: подсферы + персист + миграция.
4. SoulPanel.tsx: вкладки Записи/Журнал/Планы.
5. Вкладки Медиа + Стиль.
6. Онбординг-туториал.
7. Spark ИИ-панель.
8. Хэндофф в Тигель.
9. Чистка: старые HTML прячем за «референс», мост света остаётся.

## Принципы
- Старые HTML НЕ удаляем (канон проекта). React-Душа встаёт рядом и постепенно замещает.
- Весь свет — из общего ядра. Никаких параллельных экономик.
- 33 матрицы целиком; чакра/грахи/гуны — акцент.
