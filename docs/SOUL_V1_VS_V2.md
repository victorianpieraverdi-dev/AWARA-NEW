## Чего нет — но было в v1

### 1. Журнал опыта сферы — **высокая ценность**

Внутри сферы — таб «ЖУРНАЛ» (`renderSphJournal`, строка 6869) с типизированными записями:
- 📝 запись · 💭 переживание · 🎯 план · ✨ инсайт · 📸 фото · 🎬 видео

У каждой записи: заголовок, текст, медиа (base64 в localStorage), цепочка комментариев. Хранение — `jrn_<sphereId>` в localStorage.

### 2. Планер намерений — **высокая ценность, низкая цена**

Внутри той же вкладки «ЖУРНАЛ» — чекбокс-лист «ПЛАНЫ И НАМЕРЕНИЯ» (там же в `renderSphJournal`). Каждая строка: текст + done-чекбокс + кнопка удалить. Хранение — `plan_<sphereId>`.

### 3. Аура — **средняя ценность**

Уровень `s._auraLevel` (0..3) визуализируется кольцами вокруг сферы + текстовой меткой («○ нет / ◐ зарождается / ◑ растёт / ● полная»). Рисуется в `drawSph` ~строка 1034. В v2 сейчас близкий эффект уже даёт `tLight`, но именно «уровни» отсутствуют.

### 4. Световые монеты + поток — **средняя ценность**

`spawnLightCoins(x, y, col, amount)` — анимация монеток; `scheduleLightFlow(from, to, amount)` — поток света между сферами; `tickLightExchange()` — медленный обмен. Даёт «жизнь» полю.

### 5. Дочерние сферы (Extra) — **высокая цена**

`spawnChild(parent, seed, opt)` + вкладка «РАСШИРЕНИЕ» (`renderSphExtra`, строка 6599): за 15 монет света создаётся доп. сфера с собственным цветом, иконкой и «Нитью Резонанса» (до 3 шт). Полноценная подсистема.

### 6. Кастомизация сферы — **низкая цена**

- Переименование (`startRename`, `finishRename`, строки 6740, 6798)
- Выбор иконки (`openIconPicker`, строка 6761)
- Цвет акцента (`initSphereColorPicker`, строка 6048)

### 7. Типы связей (Link Types) — **косметика**

`LINK_TYPES` (строка 639): «резонанс / союз / конфликт / поток / зеркало» — разные цвета и подписи. В v2 все нити сейчас одинаковые золотые.

### 8. AI Spark — **нужен OpenRouter key**

`openSpark()` (строка 5684) — AI-инсайт по контексту сферы. Запрос в OpenRouter, рендер результата в `#spark-panel`.

### 9. Oracle — **нужен OpenRouter key**

`toggleOracle()` (строка 1845) — AI-чат, доступен из любой сферы. Использует `getGameContext()` для подмешивания состояния.

### 10. PsychePanel (сводный экран) — **низкая цена**

`openPsychePanel()` (строка 6102) — сводный экран по всем сферам, вкладки: обзор / сферы / свет / настройки.

### 11. Privacy / Collab — **низкий приоритет**

`openCollabSphere(parentId)` (строка 7216) + панель `#sp-pan` (строка 5330) — сфера как личная / общая / публичная, со-путники, лента, приглашения через `awara://sphere/<id>` ссылки. Соц-фича, для одиночного UX не критично.

### 12. Атмосфера / прессеты — **опционально**

`initAtmoPanel()` (строка 5965) — менять фон, цвет акцента, насыщенность звёзд. Низкий приоритет.

## Предлагаемый порядок миграции

1. **(в этом PR, T-625)** Плавный рост света — готово.
2. **Следующий PR (T-626?)** Журнал + Планер внутри сферы — даёт «зачем туда возвращаться»; тянет за собой типы записей и медиа.
3. **(T-627?)** Уровни ауры — копеечно, визуально даёт ощущение прогресса.
4. **(T-628?)** Кастомизация сферы (rename / icon / color).
5. **(T-629?)** Дочерние сферы (Extra) за свет.
6. **(T-630?)** AI Spark + Oracle (когда подключим OpenRouter key).
7. **(позже)** Типы связей, PsychePanel-обзор.
8. **(возможно никогда)** Privacy / Collab, Atmo-прессеты — если игроки попросят.

## Когда можно удалить `initiation-space.html`

После того как из v1 портированы пункты 2–4 (Журнал, Планер, Аура, Кастомизация). Privacy / Collab / Atmo — не блокеры для удаления, они социальные/декоративные.
