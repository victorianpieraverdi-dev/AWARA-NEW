# AWARA · Дорожная карта: раскрытие хаба по свету

> Как свет осознанности из Тигля постепенно размораживает и раскрывает зоны большой игры. Парный документ: `bridge-integration-plan.md`.

## Принцип-позвоночник

- **Свет — это вектор по сферам**, не одно число: ум → эго → душа → сознание → земля/храм → космос → вселенная.
- **Каждая зона раскрывается по СВОЕЙ сфере**, а не по общей сумме — метафора становится буквальной.
- **Всё раскрытие — одна функция** `zoneState(zone, vector)` над одним gate-конфигом. Невозможно рассинхронить.
- **Заморозка ≠ пустота**: космос виден силуэтом сквозь лёд — двигатель желания.
- **Честность ≠ позитив**: свет рождается и из честно прожитой боли (бонус Ардра/Златогроза).

## Gate-конфиг (источник истины — `js/light-core.js`)

| Зона | Позиция | Раскрывает сфера | Стадии |
| --- | --- | --- | --- |
| Тигель | центр | — | всегда open |
| Пространство души | слева | — | всегда open |
| Матрицы | справа | ум | locked → few → more → full+аним. |
| Даймон | снизу | душа | basic → dialogue → deeper → generations |
| Карты | сверху | сознание | locked → partial → full |
| Земля / Храм | основание | земля | locked → inner → outer |
| Космос | вверх | космос | frozen → silhouette → partial → alive |
| Вселенная | самый верх | вселенная | frozen → opening → alive |

Пороги — черновые, подбираются на игровых тестах.

## Этапы (потихоньку, каждый — отдельный и проверяемый)

- [x] **Этап 0 — Позвоночник.** `js/light-core.js`: вектор света + gate-конфиг + `zoneState()` + самопроверка. Не подключён.
- [ ] **Этап 1 — Мост тигля.** Тигель пишет «пакет света» в `state.light.spheres`. Сандбокс остаётся отдельным, формат финальный.
- [ ] **Этап 2 — Раскрытие хаба.** `zoneState()` управляет яйцом: космос заморожен-силуэт, карты/даймон/матрицы частичные.
- [ ] **Этап 3 — Душа.** `initiation-space.html` (3 сферы прошлое/настоящее/будущее) связать с записями тигля. 4-я сфера (ко-оп) — позже.
- [ ] **Этап 4 — ИИ-распределитель + голоса.** Честность/искренность/правдивость → ИИ льёт воду по сферам. Замена regex.
- [ ] **Этап 5 — Даймон-двигатель.** Реализовать `js/daimonProgression.js` по `OPUS_DAIMON_PROGRESS_PLAN.md` (уровни/чакры/стадии/грантхи).
- [ ] **Этап 6 — Прокачка матриц.** Анимация + рост матрицы от прохождений → духовная индивидуальность (сначала в сандбоксе).
- [ ] **Этап 7 — Светообмен дозирует.** За светообмен — больше диалога с даймоном / глубины матриц / генераций.
- [ ] **Этап 8 — Разморозка верха.** Космос → вселенная → карты на высоких порогах.
