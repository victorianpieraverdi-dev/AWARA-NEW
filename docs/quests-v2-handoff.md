## 1. Цель системы

Игра AWARA: 33 линзы-матрицы (уровни 1–6), 3033 квеста локаций (тиры 1–6), награды по 7 осям (discipline, compassion, clarity, will, devotion, transformation, unity), 7 типов квестов (do / observe / reflect / meditate / create / study / ritual).

До v2 выдача квестов была случайной, все квесты — однодневные, связей между линзами не было.

Квесты v2 добавляют три механики:

1. **Многодневные арки** (L5–L6): обеты/гейсы (vow), циклы наблюдения (cycle), паломничества (pilgrimage), синтез-недели (synthesis) — 3–7 дней с ежедневным чекином.
2. **Спираль и эхо архетипа**: пройденный архетип возвращается на уровень глубже (спираль); тот же архетип в другой линзе показывается как параллель (эхо).
3. **Взвешенный селектор выдачи**: профиль игрока, стихия дня, отстающая ось, спираль, эхо, анти-повтор.

## 3. Потоки данных

### Выдача обычного квеста

1. `regenDayQuests` вызывает `AwaraQuestSelector.init()` — селектор грузит индекс архетипов, арки и автопрофиль.
2. `pick(slug, maxLevel, count:3)` считает score каждого квеста: база 1.0 × профиль(1.5) × стихия дня(1.3) × отстающая ось(1.4) × спираль(1.6) × эхо(1.15) × анти-повтор(0.45, окно 10 дней).
3. Стихия квеста по типу: do→Земля, meditate→Эфир, observe/study→Воздух, reflect→Вода, create/ritual→Огонь.
4. При показе — `markServed` (питает анти-повтор), при зачёте — `markCompleted` (питает спираль и эхо).
5. Fallback: если fetch данных упал / селектор недоступен / вернул пусто — старая логика `pickDayQuests`.
6. Под квестом — одна поэтическая строка-причина (приоритет: спираль → отстающая ось → стихия → профиль → эхо; без чисел и слова score).

### Жизнь арки

1. Гейт: уровень линзы ≥ 5 и активных арок < 2 → `pickArc(slug)` предлагает арку карточкой «Многодневный путь».
2. `AwaraArcs.start(arc)` → событие `awara:arc-started`.
3. Ежедневно `checkin(arcId, note)` — максимум 1 раз в календарный день. Платит 12% суммы наград (`checkpointShare`).
4. Завершение (`doneCount ≥ duration`): платит `max(0, сумма × 1.5 − уже выплачено)` → `awara:arc-completed`.
5. Срыв (`tick()` нашёл пропусков > 1): статус `broken`, платит `max(0, сумма × доляДней × 0.4 − выплачено × 0.5)` → `awara:arc-broken`.
6. Начисление света, приоритет: `window.AwaraArcsLightAdapter(arc, amount, kind)` (реализован, зовёт `AwaraXP.processExperience` по образцу `awara-quest-flow.js` — работают множители и окна Мер) → `AwaraArcs.onLight` → фолбэк `tigel_v1.lightBonus`. **Проверка подлинности:** при чекине меняется `ladder`, а `lightBonus` НЕ растёт.
7. У паломничеств есть опциональное поле `days[]` — UI показывает задание текущего дня (`days[doneCount]`). Арки без `days` работают как раньше.

### Индекс архетипов (офлайн-пайплайн)

1. Архетип = тип квеста × мотив (мотивы — 31 регулярка по русским корням: огонь/пламя, тень/страх, служение/дар…), максимум 2 мотива на квест.
2. Ключ квеста: `slug|level|title` (для локаций `level = tier`). Селектор читает `_index.index[key].archetypes`.
3. Пересборка **ОБЯЗАТЕЛЬНА** после любого изменения текстов квестов:
   ```
   py tools/build_archetype_index.py --src data --out data/quest_archetype_index.json
   ```
   Эталон: просмотрено **5508**, размечено **5205**, архетипов **214**.
4. Паломничества:
   ```
   py tools/build_pilgrimages.py --src data --out ... --slugs ...
   ```
   по одному квесту с каждого тира 1–6, тексты чистятся, награда нормализуется к сумме 30.

## 6. Принятые продуктовые решения

1. **Строгость арок — гибкая:** 1 пропуск прощается, дальше срыв с частичным зачётом. **Не менять.**
2. **Прозрачность — поэтическая:** одна строка-причина под квестом, без чисел и слова score.
3. **Паломничества — пилот на 5 культурах** (daoist, posthuman_ai_sophianic, slavic, egyptian, celtic). Полный генератор для всех 33 — после оценки пилота.

## 7. Инварианты — НЕ НАРУШАТЬ

1. `awara-quest-arcs.js` и `awara-quest-selector.js` **не редактируются**. Вся адаптация: PATHS, AwaraArcsLightAdapter, setProfile, внешние подписки на события.
2. `data/quest_archetype_index.json` только генерируется скриптом, никогда не правится руками.
3. Ключи localStorage не переименовываются.
4. Стиль кода: **ES5, IIFE**, без сборщиков и фреймворков (никаких `=>`, `let`/`const`, template-строк в новых файлах).
5. Новые поля схемы (`duration_days`, `arc_type`, `checkpoint_prompt`, `spiral_of`, `echo_archetype`, `days`) — всегда опциональны; старые квесты и сохранения работают без изменений.
6. Скрипты лежат **в корне** репозитория (НЕ в `js/`) — рядом с `awara-quest-flow.js`, это осознанное решение.
7. `arc_quests_seed.json` — **append-only:** 17 ручных арок не трогать.
8. На Windows-машине Павла Python запускается командой **`py`** (не `python3`/`python`).
9. **Не пушить и не деплоить на Netlify** без явной команды Павла. Точечный `git add` только своих файлов — в дереве бывают параллельные правки других агентов (пример: блок «Совет 21 агента» в `tigel-app.html` — не наша работа).

## 8. Приёмка (регресс-чеклист после любых изменений)

1. `AwaraQuestSelector.init().then(function(){ return AwaraQuestSelector.pick({slug:'daoist',maxLevel:4,count:3}); })` — 3 квеста со score.
2. `start(арка)` + `checkin()` — свет через `processExperience` (`ladder` меняется, `lightBonus` — нет).
3. Reload — арка сохранена, повторный чекин в тот же день заблокирован.
4. Пропуск >1 дня — `tick()` переводит в `broken` с частичным зачётом.
5. Пересборка индекса: 5508 / 5205 / 214.
6. `?dev=arc` — обе карточки арок (обычная без блока дня, паломничество с заданием дня) + строки спирали/эха/причин; `?dev=restore` возвращает состояние точно.
7. Старый сейв (без `awara_arcs_v1` / `awara_quest_journal_v1`, а также с битым JSON) — 0 ошибок.

## 9. Известные ограничения и открытые хвосты

1. **Веса селектора и данные:**
   - **Отстающая ось (1.4) — оживлена (2026-07-21):** `awara-matrix-quests.js` → `syncSelectorProfile()` перед `pick()`: 1–2 оси с минимальной суммой из `STATE.axes` / `AwaraXP.getAxes()`, `setProfile({laggingAxes})`. Если все оси 0 или равны — lag пустой (не подделываем). Строка-причина в UI: «Ось «…» просит внимания…» (приоритет после спирали).
   - **Профильная линза (1.5)** — `autoProfile` читает `STATE.mats` и `awara_onboarding_v1`, но **экран onboarding никто не пишет** → вес почти не персонализирует. Причина-строка профиля в UI **ещё не сделана**.
2. **Следующий рекомендованный шаг:** дорогой профиль — экран профилизации на входе (занятия → линзы: чайный мастер → daoist, цифровая практика → posthuman_ai_sophianic), запись в `awara_onboarding_v1` — `autoProfile` селектора уже читает этот ключ (формат: `branches` / `maps_to_lenses` / slug…).
3. **Бэклог:** ретро-атрибуция «Тигель видит прожитое» (дизайн готов, ждёт 4 решений Павла); лексиконы 33 линз для интерпретационного языка; латание дыр покрытия (technomagical, aztec_mexica; архетипы сострадания/единства); курация 214 архетипов до 150–200 канонических (+285 неразмеченных квестов); полный генератор паломничеств на 33 культуры после оценки пилота.
4. **Крайний случай tick():** вкладка, открытая через полночь без перезагрузки, не вызовет `tick()` до следующего открытия. Признано некритичным, **не чинить**.
